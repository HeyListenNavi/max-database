Lógica del Algoritmo de Generación de Horarios
El sistema utiliza un algoritmo de selección secuencial que organiza las materias del alumno basándose en la prioridad académica y la eficiencia del tiempo. El proceso se divide en tres etapas principales:
1. Priorización de Materias (Ordenamiento)
El primer paso consiste en decidir qué materias son más importantes para el alumno. El sistema asigna un nivel de importancia a cada materia disponible basándose en tres criterios:
Selección del Usuario: Si el alumno marcó la materia como prioritaria, esta se coloca al principio de la lista.
Materias Reprobadas: Se da preferencia alta a las materias que el alumno ha reprobado anteriormente para asegurar que las curse nuevamente lo antes posible.
Nivel del Semestre: Se priorizan las materias de semestres inferiores sobre las avanzadas para evitar que el alumno se rezague en su plan de estudios.
2. Validación de Requisitos (Filtro)
Una vez ordenada la lista de materias por importancia, el sistema intenta asignar cada una al horario, verificando que cumpla con tres condiciones obligatorias:
Seriación: Comprueba si el alumno ya aprobó las materias previas necesarias (prerrequisitos).
Disponibilidad: Verifica que el grupo tenga cupo disponible (capacidad mayor a cero).
Conflicto de Horario: Revisa que la hora de la clase no se empalme con ninguna otra materia que ya haya sido agregada al horario.
3. Selección del Mejor Grupo (Optimización)
Si una materia tiene varios grupos disponibles (por ejemplo, Grupo A y Grupo B) y ambos cumplen con los requisitos anteriores, el sistema debe decidir cuál elegir.
Para esto, utiliza un criterio de continuidad:
Calcula cuánto tiempo libre (horas muertas) quedaría entre la nueva clase y las clases ya agendadas.
Selecciona el grupo que genere menos huecos en el horario, buscando que la jornada del estudiante sea lo más compacta y continua posible.
